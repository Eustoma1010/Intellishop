🎯 HOW IT WORKS
==============================================================================

User Flow:
1. User clicks "Đăng Nhập"
2. User clicks Google button
3. Google popup shows
4. User selects Gmail account
5. Popup closes
6. handleGoogleCallback() executed
7. processSocialPayload() sends to backend
8. Backend checks: email exists?
   - YES → Return JWT tokens (login existing user)
   - NO → Return temp_token (show registration form)
9. Frontend saves JWT tokens
10. User redirected to home page ✅

Backend Endpoints:
- POST /api/social-check/
  → Checks if user exists, returns action
- POST /api/social-complete/
  → Creates new user, returns JWT tokens

Database:
- User created with email from Google
- SocialAccount linked to user
- JWT tokens stored in browser localStorage
- User can logout and login again

==============================================================================
⚡ COMMON ISSUES & FIXES
==============================================================================

Issue 1: "Origin not allowed" error
Fix: Add localhost URLs to Google Cloud Console authorized origins

Issue 2: "Domain mismatch" error
Fix: Update Django SITES domain to 'localhost:8000' in admin

Issue 3: Google button not showing
Fix: Check Client ID in HTML, clear browser cache

Issue 4: Email not being sent
Fix: Grant email permission in Google popup

Issue 5: Callback URL error
Fix: Check redirect URI in Google Cloud Console

See documentation files for detailed troubleshooting.

==============================================================================
🚀 PRODUCTION DEPLOYMENT
==============================================================================

When deploying to production:

1. Google Cloud Console
   - Update authorized JavaScript origins
   - Update authorized redirect URIs
   - Get production credentials

2. Backend .env
   FRONTEND_URL=https://intelishop-frontend.vercel.app
   GOOGLE_OAUTH_CLIENT_ID=production-id
   GOOGLE_OAUTH_CLIENT_SECRET=production-secret

3. Django Admin
   - Update SITES domain to production URL
   - Update Social Application credentials

4. Django Settings
   DEBUG=False
   ALLOWED_HOSTS=intelishop-backend.onrender.com

5. Frontend
   - Update Client ID in HTML (if hardcoded)
   - Or use environment variable
