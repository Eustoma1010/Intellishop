🔄 THE 4 FLOWS AT A GLANCE

REGISTRATION:

1A (Local Form)
  Email + Password + Name + Phone
  → Validation & Hash
  → Create User
  → Auto-login ✅

1B (Google New User)
  Google Account
  → Email not exist?
  → Show Scenario 2 form
  → Collect Phone + Address
  → Create User + SocialAccount
  → Auto-login ✅

LOGIN:

2A (Local)
  Email + Password
  → Validate Email exist
  → Check has password
  → Verify password
  → JWT tokens ✅

2B (Google)
  Google Account
  → Email not exist? → Go to 1B
  → Email exists:
     - Already linked? → Login ✅
     - Not linked? → Auto-link + Login ✅

📊 API ENDPOINTS (UPDATED)

POST /api/register/
  Request: {email, password, name, phone, address?}
  Response: {success, access_token, refresh_token, user}
  Errors: 400, 500

POST /api/login/
  Request: {email, password}
  Response: {success, access_token, refresh_token, user}
  Errors: 404, 400, 401, 500

POST /api/social-check/
  Request: {email, name, provider, uid}
  Response:
    - action: "login" → {access_token, refresh_token, user}
    - action: "requires_info" → {temp_token, email, name}
  Errors: 400, 500

POST /api/social-complete/
  Request: {temp_token, phone?, address?, password?}
  Response: {success, access_token, refresh_token, user}
  Errors: 400, 409, 500
